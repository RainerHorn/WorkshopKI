Code-Unterstützung mit Python

Ein lokales LLM als Programmierhilfe – Skripte generieren, erklären und verbessern lassen.

🎯 Aufgabe
  • Das lokale LLM ein Python-Skript zur Textanonymisierung schreiben lassen
  • Den generierten Code verstehen, testen und schrittweise verbessern
  • Lernen, wie man iterativ mit einem LLM programmiert: generieren → testen → nachfragen
💡 Voraussetzungen
  • Python 3.x auf dem System installiert (python --version im Terminal prüfen)
  • Ein laufendes lokales Modell – empfohlen: llama3.1:8b oder phi3:mini
  • Kein Python-Vorwissen nötig – das Modell erklärt den Code auf Wunsch

Schritt für Schritt

1

Skript generieren lassen

Schicken Sie den folgenden Prompt an Ihr lokales Modell. Das Modell soll ein fertiges, lauffähiges Python-Skript ausgeben.

Schreibe ein Python-Skript, das einen Text aus einer Datei liest und alle personenbezogenen Daten durch Platzhalter ersetzt. Anforderungen: - Eingabe: eine Textdatei (eingabe.txt) - Ersetze Vor- und Nachnamen durch [NAME] - Ersetze Datumsangaben (TT.MM.JJJJ) durch [DATUM] - Ersetze E-Mail-Adressen durch [EMAIL] - Ausgabe: neue Datei (ausgabe.txt) mit den ersetzten Inhalten - Das Skript soll kommentiert sein, damit man es leicht versteht - Nutze nur die Python-Standardbibliothek (kein pip install nötig) Gib ausschließlich den Python-Code aus, keinen erklärenden Text drumherum.
2

Code in eine Datei speichern

Kopieren Sie den generierten Code, öffnen Sie einen Texteditor (z. B. Notepad, VS Code) und speichern Sie ihn als anonymisieren.py.

Legen Sie außerdem eine eingabe.txt mit folgendem Testinhalt an:

Testdatei – eingabe.txt
Sehr geehrte Frau Musterfrau, wie am 14.04.2026 besprochen, übersenden wir Ihnen die Unterlagen für Herrn Max Mustermann (geb. 03.09.2008). Für Rückfragen erreichen Sie uns unter schulleitung@bbs-musterstadt.de oder über das Sekretariat. Frau Anna Bergmann steht Ihnen werktags von 08:00 bis 13:00 Uhr zur Verfügung. Mit freundlichen Grüßen Thomas Koch Klassenleitung KFZ 12c
3

Skript ausführen

Öffnen Sie ein Terminal im selben Ordner wie die Dateien und führen Sie aus:

Terminal-Befehl
python anonymisieren.py

Prüfen Sie danach die erzeugte ausgabe.txt: Wurden alle Namen, Daten und E-Mail-Adressen ersetzt?

4

Fehler und Lücken melden – iterativ verbessern

Wahrscheinlich funktioniert nicht alles auf Anhieb. Beschreiben Sie dem Modell, was fehlt oder falsch ist – und lassen Sie es den Code korrigieren.

Was das LLM kann – und wo es scheitert

Aufgabe LLM-Eignung Warum?
Einfaches Skript aus Anforderungen generieren ✅ Sehr gut Klare Aufgabe, bekanntes Muster
Fehlermeldung erklären und beheben ✅ Gut Fehlermeldungen sind eindeutig und kontextreich
Unbekannte Namen zuverlässig erkennen ⚠️ Schwach Keine NER-Fähigkeit ohne spezielles Modell
Code auf Sicherheitslücken prüfen ⚠️ Bedingt Offensichtliche Fehler ja – subtile nein
Komplexe, mehrstufige Algorithmen ❌ Unsicher Ergebnis immer manuell testen

Zusatzaufgaben

➕ Wenn Sie schneller fertig sind
  • Das Skript so erweitern, dass es alle .txt-Dateien in einem Ordner auf einmal verarbeitet
  • Eine Liste bekannter Namen aus einer separaten Datei einlesen und gezielt ersetzen lassen
  • Das Skript in eine kurze Funktion umbauen und mit dem Modell Unit-Tests dafür generieren lassen
  • Dasselbe Skript in einer anderen Sprache generieren lassen (z. B. Bash oder JavaScript) – Vergleich der Ergebnisse
