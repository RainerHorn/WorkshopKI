MCP – Lokale LLMs mit Werkzeugen verbinden

Das Modell bekommt Zugriff auf Echtzeit-Daten, Dokumentationen und das Web – vollständig lokal steuerbar.

🎯 Aufgabe
  • Verstehen, was MCP ist und wie es ein LLM mit externen Werkzeugen verbindet
  • Beispiel 1: Einen einfachen lokalen Zeitserver als MCP-Tool einrichten und befragen
  • Beispiel 2: Context7 als MCP-Server nutzen – aktuelle Bibliotheksdokumentationen abfragen
  • Beispiel 3: Websuche per MCP – das lokale Modell recherchiert im Internet
📖 Was ist MCP? Das Model Context Protocol (MCP) ist ein offener Standard von Anthropic (2024), der definiert, wie ein LLM mit externen Werkzeugen – sogenannten MCP-Servern – kommuniziert. Statt Daten nur aus dem Trainings-Wissen zu ziehen, kann das Modell aktiv Werkzeuge aufrufen: Uhrzeiten abfragen, Code ausführen, Webseiten laden, Datenbanken abfragen. Open WebUI unterstützt MCP ab Version 0.5.

Beispiel 1: Lokaler Zeitserver

Ein einfacher Python-MCP-Server, der dem Modell Datum und Uhrzeit bereitstellt

Das Modell kennt von sich aus keine aktuelle Uhrzeit. Mit einem MCP-Zeitserver kann es diese Information jederzeit abrufen – ein ideales erstes Beispiel, weil es keine externe Abhängigkeit hat.

1

MCP-Paket installieren

Terminal
pip install mcp
2

Zeitserver-Skript erstellen

Speichern Sie folgenden Code als zeitserver.py:

zeitserver.py
from mcp.server.fastmcp import FastMCP from datetime import datetime import pytz mcp = FastMCP("Zeitserver") @mcp.tool() def get_current_time(timezone: str = "Europe/Berlin") -> str: """Gibt die aktuelle Uhrzeit und das Datum für eine Zeitzone zurück.""" try: tz = pytz.timezone(timezone) now = datetime.now(tz) return now.strftime(f"Datum: %d.%m.%Y | Uhrzeit: %H:%M:%S | Zeitzone: {timezone}") except Exception: return f"Unbekannte Zeitzone: {timezone}" @mcp.tool() def get_weekday(timezone: str = "Europe/Berlin") -> str: """Gibt den aktuellen Wochentag zurück.""" tz = pytz.timezone(timezone) now = datetime.now(tz) tage = ["Montag","Dienstag","Mittwoch","Donnerstag","Freitag","Samstag","Sonntag"] return tage[now.weekday()] if __name__ == "__main__": mcp.run(transport="stdio")
Terminal – pytz installieren
pip install pytz
3

MCP-Server in Open WebUI registrieren

In Open WebUI: Einstellungen → Werkzeuge → MCP → + Neuer Server
Tragen Sie folgende Konfiguration ein:

MCP-Konfiguration (Open WebUI)
{ "name": "Zeitserver", "transport": "stdio", "command": "python", "args": ["C:/Pfad/zu/zeitserver.py"] }

Den Pfad zu zeitserver.py entsprechend anpassen.

4

Prompts testen

A

Einfache Zeitabfrage

Das Modell soll das Tool automatisch aufrufen

Wie spät ist es gerade in Berlin? Und was ist heute für ein Wochentag?
B

Zeitzonen vergleichen

Mehrere Tool-Aufrufe in einer Anfrage

Wie viel Uhr ist es gleichzeitig in Berlin, New York (America/New_York) und Tokio (Asia/Tokyo)? Zeige die drei Zeiten als übersichtliche Tabelle.
💡 Was passiert im Hintergrund? In Open WebUI sehen Sie unter der Antwort ein kleines Werkzeug-Symbol. Klicken Sie darauf – Sie sehen genau, welches Tool das Modell aufgerufen hat und was der Server zurückgegeben hat. So können Sie die Denkschritte des Modells transparent nachvollziehen.

Beispiel 2: Context7 – aktuelle Dokumentationen

Das Modell liest Bibliotheks-Docs direkt nach – keine veralteten Trainingsdaten mehr

Ein LLM kennt nur Inhalte aus seinem Trainings-Datensatz – Bibliotheken wie Python-Pakete oder Frameworks ändern sich aber ständig. Context7 ist ein MCP-Server, der auf Anfrage die aktuelle offizielle Dokumentation für Tausende von Bibliotheken abruft und dem Modell als Kontext bereitstellt.

1

Context7 als MCP-Server einrichten

Context7 läuft als npm-Paket und benötigt eine Internetverbindung.

Terminal – einmalige Installation prüfen
npx -y @upstash/context7-mcp
2

In Open WebUI registrieren

MCP-Konfiguration (Open WebUI)
{ "name": "Context7", "transport": "stdio", "command": "npx", "args": ["-y", "@upstash/context7-mcp"] }
3

Context7 im Prompt aktivieren

Context7 wird automatisch aktiv, wenn Sie use context7 in Ihren Prompt schreiben. Das Modell sucht dann zuerst nach der passenden Bibliotheksdokumentation.

A

Python-Dokumentation abfragen

Aktuelle Syntax aus den offiziellen Docs

use context7 Wie lese ich eine CSV-Datei mit pandas ein und filtere Zeilen nach einem Spaltenwert? Zeige ein vollständiges Beispiel mit der aktuellen pandas-API.
B

Framework-Dokumentation

Aktuelle Informationen zu einem Webframework

use context7 Zeige mir, wie ich in FastAPI einen POST-Endpunkt mit Pydantic-Validierung erstelle. Nutze die aktuelle Dokumentation.
C

Vergleich: mit und ohne Context7

Den Unterschied direkt sehen

use context7 Wie installiere ich Ollama per Python SDK und sende eine einfache Chat-Anfrage? Ich möchte die aktuelle API-Syntax aus der offiziellen Dokumentation.
🔬 Beobachtungsaufgabe Stellen Sie denselben Code-Prompt einmal mit und einmal ohne use context7. Vergleichen Sie:
  • Ist die zurückgegebene API-Syntax identisch?
  • Welche Version der Bibliothek referenziert das Modell?
  • Wie verhält sich das Modell, wenn die Bibliothek sehr neu ist?

Beispiel 3: Websuche per MCP

Das lokale Modell recherchiert im Internet – Ergebnisse bleiben auf Ihrem Rechner

Mit einem Such-MCP-Server kann das Modell aktuelle Informationen aus dem Web abrufen. Die Suchergebnisse werden lokal verarbeitet – keine Daten gelangen an externe KI-Dienste. Hier verwenden wir Brave Search MCP (kostenloser API-Key) und als Alternative den DuckDuckGo MCP-Server ohne API-Key.

⚠️ Datenschutz-Hinweis Die Suchanfragen selbst gehen an den Suchanbieter (Brave bzw. DuckDuckGo). Die Verarbeitung der Ergebnisse findet vollständig lokal statt. Keine personenbezogenen Daten in Suchanfragen eingeben.

Option A: DuckDuckGo (kein API-Key nötig)

1

DuckDuckGo MCP-Server einrichten

MCP-Konfiguration (Open WebUI)
{ "name": "DuckDuckGo Suche", "transport": "stdio", "command": "uvx", "args": ["duckduckgo-mcp-server"] }
Terminal – uv installieren (falls nicht vorhanden)
pip install uv

Option B: Brave Search (höhere Qualität, API-Key erforderlich)

1

Kostenlosen API-Key holen

Registrierung unter brave.com/search/api – Free Tier: 2.000 Anfragen/Monat, keine Kreditkarte nötig.

2

Brave MCP-Server konfigurieren

MCP-Konfiguration (Open WebUI)
{ "name": "Brave Suche", "transport": "stdio", "command": "npx", "args": ["-y", "@modelcontextprotocol/server-brave-search"], "env": { "BRAVE_API_KEY": "IHR-API-KEY-HIER" } }
3

Websuche-Prompts testen

A

Aktuelle Neuigkeit

Eine Information, die das Modell nicht kennen kann

Suche im Web nach aktuellen Neuigkeiten zu lokalen Sprachmodellen (LLM) aus den letzten 7 Tagen. Fasse die drei wichtigsten Meldungen auf Deutsch zusammen.
B

Faktenfrage mit Quellenangabe

Überprüfbare Recherche mit Belegen

Suche im Web: Welche KI-Regelungen gelten aktuell für Schulen in Deutschland laut dem EU AI Act? Gib die Quelle (URL) mit an und erkläre die wichtigsten Punkte in einfacher Sprache.
C

Rechercheaufgabe für den Unterricht

Aktueller Anlass für eine Unterrichtsstunde

Suche im Web nach einem aktuellen Nachrichtenartikel über ein technisches Thema, das für Auszubildende im IT-Bereich relevant ist. Fasse den Artikel in 5 Stichpunkten zusammen und formuliere zwei Lernfragen daraus, die sich für den Unterrichtseinstieg eignen.
