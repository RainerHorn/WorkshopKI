1. Voraussetzungen

Mindestanforderungen
  • Windows 10/11 (64-bit), macOS 12+ oder Linux
  • Mindestens 16 GB RAM (32 GB empfohlen für mittlere Modelle)
  • Ca. 15–20 GB freier Speicherplatz für Modelle
  • Internetzugang für den Download (einmalig)
GPU-Beschleunigung (optional)
  • NVIDIA-GPU mit ≥ 6 GB VRAM → CUDA wird automatisch erkannt
  • AMD-GPU (Linux) → ROCm-Unterstützung
  • Apple Silicon (M1/M2/M3) → Metal-Beschleunigung integriert
  • Ohne GPU läuft Ollama auf der CPU – langsamer, aber vollständig nutzbar
⚠️ Docker für Open WebUI Open WebUI wird über Docker betrieben. Installieren und starten Sie Docker Desktop, bevor Sie mit Abschnitt 5 beginnen.

Download: docker.com/products/docker-desktop

2. Ollama installieren

1

Installer herunterladen und ausführen

Laden Sie Ollama von der offiziellen Website herunter:

👉 ollama.com/download

Betriebssystem wählen und Installer ausführen. Ollama startet anschließend automatisch als Hintergrunddienst.

2

Installation prüfen

Terminal öffnen (PowerShell / Terminal / Bash) und eingeben:

ollama --version

Erwartete Ausgabe: ollama version 0.x.x

3

Dienststatus prüfen (optional)

ollama serve

Falls Ollama bereits läuft, erscheint eine Meldung. Abbrechen mit Strg+C. Normalerweise läuft der Dienst automatisch im Hintergrund.

3. Modelle herunterladen

Empfohlene Modelle für den Workshop – sortiert nach Größe:

Klein · ~2 GB

Phi-3 Mini

Microsofts kompaktes Modell. Sehr schnell, läuft problemlos auf CPU.

3,8 B Parameter · 4-bit quantisiert
Klein · ~2 GB

Llama 3.2 (3B)

Metas kleines Modell. Gute Sprachqualität, auch auf Deutsch.

3 B Parameter · 4-bit quantisiert
Mittel · ~5 GB

Llama 3.1 (8B)

Solides Allround-Modell. Braucht ~8 GB RAM, profitiert von GPU.

8 B Parameter · 4-bit quantisiert
Mittel · ~5 GB

Mistral 7B

Sehr gute Leistung pro Größe. Ideal für strukturierte Ausgaben.

7 B Parameter · 4-bit quantisiert
4

Kleine Modelle laden

ollama pull phi3:mini
ollama pull llama3.2:3b
5

Mittlere Modelle laden

ollama pull llama3.1:8b
ollama pull mistral:7b
6

Alle geladenen Modelle anzeigen

ollama list

4. Test in der Konsole

Ollama bringt eine eingebaute Chat-Shell mit. Modelle lassen sich direkt im Terminal testen – ohne Browser oder weitere Software.
7

Modell starten und chatten

Kleines Modell starten:

ollama run phi3:mini

Mittleres Modell starten:

ollama run llama3.1:8b

Nach dem Start erscheint >>>. Frage eintippen → Enter. Beenden: /bye oder Strg+D.

🧪 Test-Prompts für die Konsole Diese Prompts direkt in die Ollama-Eingabeaufforderung eingeben:

Einstiegstest:

Erkläre mir in drei Sätzen, was ein Large Language Model ist.

Schlechter Prompt (zum Vergleich):

Schreib was über Datenschutz.

Strukturierter Prompt (gleiche Frage, besser formuliert):

Du bist ein Datenschutzbeauftragter an einer Berufsbildenden Schule. Fasse in 5 Stichpunkten zusammen, welche personenbezogenen Daten von Schülerinnen und Schülern besonders schützenswert sind.
💡 Tipp: Führen Sie denselben strukturierten Prompt mit phi3:mini und llama3.1:8b aus und vergleichen Sie Qualität, Länge und Stil der Antworten.

5. Open WebUI installieren & starten

Open WebUI ist eine browserbasierte Chat-Oberfläche, die sich direkt mit dem lokalen Ollama-Dienst verbindet. Alle Daten bleiben auf Ihrem Rechner.
8

Docker Desktop starten

Stellen Sie sicher, dass Docker Desktop läuft (Icon in der Taskleiste). Status prüfen:

docker info

Alles in Ordnung, wenn keine „Cannot connect"-Meldung erscheint.

9

Open WebUI starten

Windows & Linux (Ollama läuft lokal):

docker run -d -p 3000:8080 \ --add-host=host.docker.internal:host-gateway \ -v open-webui:/app/backend/data \ --name open-webui --restart always \ ghcr.io/open-webui/open-webui:main

macOS (Ollama läuft lokal):

docker run -d -p 3000:8080 \ -e OLLAMA_BASE_URL=http://host.docker.internal:11434 \ -v open-webui:/app/backend/data \ --name open-webui --restart always \ ghcr.io/open-webui/open-webui:main
⚠️ Erster Start: Das Docker-Image (~1 GB) wird heruntergeladen – das kann einige Minuten dauern. Fortschritt verfolgen:
docker logs -f open-webui
10

Open WebUI im Browser öffnen

Sobald der Container läuft, im Browser aufrufen:

http://localhost:3000

Beim ersten Aufruf lokalen Admin-Account anlegen (Name, E-Mail, Passwort). Die Daten bleiben lokal auf Ihrem Rechner.

11

Container stoppen & neu starten

docker stop open-webui
docker start open-webui

6. Test in Open WebUI

Open WebUI verbindet sich automatisch mit Ollama und zeigt alle geladenen Modelle an. Das Modell lässt sich jederzeit über das Dropdown oben im Chat wechseln.
12

Erstes Gespräch starten

  1. Oben links das Modell-Dropdown öffnen
  2. Modell wählen, z. B. phi3:mini oder llama3.1:8b
  3. Prompt eingeben und absenden
🧪 Test-Prompts für Open WebUI

Zusammenfassung mit Rollenanweisung:

Du bist eine erfahrene Lehrkraft an einer Berufsschule. Fasse den folgenden Text für Schülerinnen und Schüler in 5 Stichpunkten zusammen. Verwende einfache, klare Sprache. [Hier Ihren eigenen Text einfügen]

Feedback-Entwurf (pseudonymisiert):

Erstelle einen sachlichen, wertschätzenden Feedback-Text für eine Schülerin (Pseudonym: „S1"), die im Praktikum folgende Kompetenzen gezeigt hat: Pünktlichkeit, eigenständiges Arbeiten, gute Kommunikation mit Kunden. Der Text soll 4–5 Sätze umfassen und als Vorlage für ein Beurteilungsschreiben dienen.

Datenschutz-Vergleich (mit zwei Modellen testen):

Welche drei größten Datenschutzrisiken entstehen, wenn Lehrkräfte Schülerdaten in Cloud-KI-Dienste eingeben? Antworte präzise in drei nummerierten Punkten.
💡 Arena-Modus: Open WebUI unterstützt ab Version 0.3 einen Vergleichsmodus, in dem zwei Modelle denselben Prompt gleichzeitig beantworten. Im Chat oben auf „+" klicken und ein zweites Modell hinzufügen.
✅ Checkliste: Alles erfolgreich installiert?
  • ☐  ollama --version zeigt eine Versionsnummer
  • ☐  ollama list zeigt mindestens zwei Modelle
  • ☐  ollama run phi3:mini antwortet auf eine Eingabe
  • ☐  http://localhost:3000 öffnet die Open-WebUI-Oberfläche
  • ☐  Im WebUI-Chat ist ein Modell auswählbar und antwortet
🔒 Datenschutz-Hinweis Alle Modelle laufen ausschließlich lokal – keine Daten werden an externe Server übertragen. Dennoch gilt: Echtnamen und direkte Personenbezüge (z. B. Schülernamen, Geburtsdaten) durch Pseudonyme ersetzen, bevor Texte ins Modell eingegeben werden.
