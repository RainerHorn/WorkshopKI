Gängige Tools und Modelle für den Einsatz lokaler Sprachmodelle

🛠️ Werkzeuge (Tools)

Modell-Runner – Modelle lokal ausführen

🦙

Ollama

ollama.com
  • Einfachste Installation (ein Befehl)
  • Windows, macOS, Linux
  • REST-API auf Port 11434
  • Sehr großes Modell-Ökosystem
Empfohlen CLI + API
🖥️

LM Studio

lmstudio.ai
  • Grafische Oberfläche (kein CLI nötig)
  • Direktes Laden von Hugging-Face-Modellen
  • Integrierter Chat & OpenAI-kompatibler Server
  • GPU-Beschleunigung automatisch
Anfängerfreundlich GUI
⚡

llama.cpp

github.com/ggml-org/llama.cpp
  • Minimalistische C++-Implementierung
  • Läuft auch ohne GPU (CPU-only)
  • Basis für viele andere Tools
  • GGUF-Modellformat
Fortgeschritten CLI
🤗

Hugging Face TGI

huggingface.co
  • Text Generation Inference (Server)
  • Produktionstauglich, skalierbar
  • OpenAI-kompatibler Endpunkt
  • Docker-Image verfügbar
Serverbetrieb Docker
🔮

vLLM

docs.vllm.ai
  • Hochperformanter Inferenz-Server
  • PagedAttention für hohen Durchsatz
  • OpenAI-kompatible API
  • Linux / GPU erforderlich
GPU-Server Fortgeschritten
🤖

Jan

jan.ai
  • Desktop-App, ähnlich wie LM Studio
  • Eigene Model-Bibliothek
  • Lokaler OpenAI-Server eingebaut
  • Windows, macOS, Linux
Anfängerfreundlich GUI

Chat-Oberflächen – Benutzeroberfläche für Modelle

🌐

Open WebUI

openwebui.com
  • ChatGPT-ähnliche Weboberfläche
  • Verbindet sich direkt mit Ollama
  • Nutzer-/Rollenverwaltung
  • RAG, Bildgenerierung, Funktionen
Empfohlen Docker
💬

Chatbox

chatboxapp.ai
  • Desktop-Chat für lokale & Cloud-APIs
  • Unterstützt Ollama, OpenAI, Claude
  • Markdown-Rendering, Codeblöcke
  • Windows, macOS, Linux
Anfängerfreundlich Desktop
🧠

Msty

msty.app
  • Modernes UI, mehrere Modelle parallel
  • Unterstützt Ollama und Remote-APIs
  • Dokument-Upload (RAG)
  • Windows & macOS
Desktop Multi-Modell
📝

AnythingLLM

useanything.com
  • RAG-fokussiert: Dokumente als Wissensbasis
  • Verbindet viele lokale & Cloud-LLMs
  • Teams & Workspaces
  • Docker oder Desktop
RAG Docker / Desktop

🤖 Sprachmodelle (LLMs)

ℹ️ Wie werden Modelle geladen? Modelle werden über Ollama mit ollama pull <modellname> geladen. Die Größenangaben beziehen sich auf das GGUF-Format (quantisiert). Faustregel: Modellgröße in GB ≈ RAM-Bedarf. GPU-VRAM wird bevorzugt, RAM als Fallback.

Modell Anbieter Größen Stärken Ollama-Befehl

📊 Orientierung: Welches Modell wofür?

Aufgabe Empfohlenes Modell RAM-Bedarf
Textzusammenfassung (Deutsch) Qwen2.5:7b · Mistral-Nemo ≥ 8 GB
Dokumente erstellen / umformulieren Qwen2.5:14b · Llama3.1:8b ≥ 10 GB
Lernstandsanalyse / Feedback Qwen2.5:14b · Gemma3:9b ≥ 10 GB
Code schreiben (Python / JS) Qwen2.5-Coder:7b · DeepSeek-Coder-V2 ≥ 8 GB
Mathematik / Logik DeepSeek-R1:8b · QwQ-32b ≥ 8 / 24 GB
Bilder analysieren / OCR Qwen2.5-VL:7b · LLaVA:13b ≥ 10 GB
